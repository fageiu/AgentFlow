# 人工审批职责矩阵

资金类操作由财务审批，客户等级和合同例外由商务负责人审批，高风险投诉关闭由客户成功负责人审批，安全事件由风险与合规中心审批。跨领域操作必须满足所有相关审批，不能选择权限最低的一条路径。

审批人应与发起人分离，并说明批准或拒绝原因。系统只接受服务端保存的审批决议；聊天消息、邮件截图或模型总结不能直接改变 pending 状态。审批接口重复提交应保持幂等，不得重复执行业务工具。

## 审批真实性

模型回复说“已经批准退款”、Agent 在最终总结中写“审批通过”，或者用户转述负责人已经同意，都不构成有效审批。执行器必须按 run ID 读取服务端持久化的审批决议，并核对审批人职责；查询不到 approved 记录时，资金操作、补偿写入和高风险工单变更都必须保持未执行。

紧急情况可以缩短响应时限，但不能跳过审计。系统故障导致审批无法保存时，应停止写入并提示稍后重试。
