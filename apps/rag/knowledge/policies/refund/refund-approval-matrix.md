# 退款审批权限矩阵

## 基本原则

退款属于高风险资金操作，不论客户等级和金额大小，都必须保留人工审批记录。客户等级只影响受理优先级，不改变审批要求。审批人必须能够看到工单、客户、订单、适用政策、退款金额和原因，不得批准缺少订单依据的申请。

## 分级权限

- 5000 元及以下：客户成功主管审批。
- 5000 元以上至 50000 元：客户成功主管和财务经理共同审批。
- 50000 元以上：财务负责人审批，并由法务确认合同责任。
- 高风险客户、异常支付或跨主体收款：无论金额均转交财务风控。

发起人不得作为唯一审批人。审批拒绝后，系统应停止退款创建和后续工单状态写入；如需重新提交，必须补充材料并创建新的审批决议，不得覆盖旧记录。

## 系统控制

Agent 可以完成事实核查、生成建议和提交审批请求，但不能代替审批人。审批接口应记录 run ID、工具调用 ID、审批人、决议、时间和原因。模型自然语言中的“已批准”不能作为事实，只有服务端审批状态才是可信依据。
