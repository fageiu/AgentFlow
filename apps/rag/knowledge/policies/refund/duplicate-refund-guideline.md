# 重复退款申请核查指引

## 识别范围

同一订单存在待审批、已创建或处理中退款时，再次提出退款即视为潜在重复申请。工单标题不同、申请渠道不同或客户更换联系人，不改变同一订单只能存在一条有效退款流程的原则。

## 核查步骤

客服应读取订单退款状态、退款记录、关联工单和上一轮审批结论。已有待审批记录时应复用原记录并告知当前进度；已有完成退款时不得再次创建；上一轮被拒绝且客户补充了新证据时，可以重新进入人工评估，但必须引用原审批记录。

## 幂等要求

退款工具必须返回 created 或 reused 等操作元数据。调用次数不能直接解释为实际写入次数。并发任务处理同一订单时，应通过幂等键或事务锁阻止双重创建。发现状态不一致时停止自动处理，转交人工核对，不得通过重复调用尝试“修复”。
